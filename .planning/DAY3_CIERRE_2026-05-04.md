## Endpoints canónicos disponibles

### Lectura / auditoría
- `GET  /api/loanbook` — lista con próxima cuota canónica + flag vencida
- `GET  /api/loanbook/{id}` — detalle con cronograma completo
- `GET  /api/loanbook-admin/motor/audit-all` — semáforo verde/amarilla/roja

### Operación diaria
- `PUT  /api/loanbook/{id}/entrega` — activa crédito + genera cronograma canónico
- `PUT  /api/loanbook/{id}/pago` — registra pago con waterfall §4.1
- `PUT  /api/loanbook/{id}/pago/inicial` — registra pago de cuota 0 sin waterfall

### Admin / mantenimiento
- `POST /api/loanbook-admin/motor/migrar?dry_run` — recalcula derivados de todos
- `POST /api/loanbook-admin/aplicar-patches?dry_run` — bulk patch desde JSON
- `POST /api/loanbook-admin/inyectar-cuota-inicial?dry_run` — agrega cuota 0 al cronograma
- `POST /api/loanbook-admin/marcar-cuotas-iniciales-pagadas?dry_run` — marca cuota 0 pagada
- `POST /api/loanbook-admin/regenerar-cronograma-lb?dry_run` — regenera cronograma de un LB

### Endpoints DEPRECATED (mantienen funcionalidad legacy)
- `POST /api/loanbook/{id}/registrar-pago` → migrar a `PUT /pago`
- `POST /api/loanbook/{id}/registrar-pago-inicial` → migrar a `PUT /pago/inicial`

### B2.8 — `cuotas_vencidas` cuenta cuota 0 vencida
En `motor.derivar_estado`, el conteo `cuotas_vencidas` incluye cuota 0.
La política dice cuota 0 vencida no genera mora ni delinquency, pero sí
aparece en `cuotas_vencidas`. No afecta DPD/estado/sub_bucket. Fix: agregar
guard `is_cuota_inicial` en el loop de cuotas_vencidas.

Workaround: correr `motor/migrar` después.
