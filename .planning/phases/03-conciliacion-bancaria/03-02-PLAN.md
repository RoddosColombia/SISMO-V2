  <action>
Create backend/agents/contador/handlers/conciliacion.py with 5 handlers:

1. handle_conciliar_extracto_bancario: 
   - Receives file_path + banco (auto-detected)
   - Anti-dup Capa 1: check extract hash
   - Parse with appropriate parser
   - For each movement: anti-dup Capa 2, classify with confidence
   - >= 0.70: queue for auto-cause (request_with_verify + publish_event)
   - < 0.70: insert into backlog_movimientos
   - Use BackgroundTasks for > 10 movements, store job in conciliacion_jobs
   - Return: {total, causados, backlog, duplicados, job_id}

2. handle_clasificar_movimiento:
   - Single movement classification with confidence 0-1
   - Extend _classify_gasto pattern from egresos.py
   - Special patterns: "GRAVAMEN" → impuesto 1.0, socio CC → CXC 1.0
   - Return: {cuenta_sugerida, confianza, tipo, retenciones}

3. handle_enviar_movimiento_backlog:
   - Insert movement into backlog_movimientos (MongoDB operational, allowed)
   - Include: fecha, banco, descripcion, monto, razon_pendiente, intentos=0

4. handle_causar_desde_backlog:
   - Read movement from backlog, build journal, request_with_verify
   - On success: update backlog estado="causado", publish event
   - On failure: update backlog estado="error" + razon

5. handle_consultar_movimientos_pendientes:
   - Query backlog_movimientos with filters (banco, fecha, limit)
   - Read-only, no confirmation needed
  </action>
