    Constants: POLL_INTERVAL_SECONDS=5, MAX_RETRIES=3, BATCH_SIZE=50

    class EventProcessor:
      __init__(self, db: AsyncIOMotorDatabase):
        - self.db, self._handlers (dict[str, list[dict]]), self._running=False, self._events_processed=0

      register(self, event_type: str, handler_fn, critical: bool = False):
        - Append {"fn": handler_fn, "critical": critical} to self._handlers[event_type]

      async _get_cursor_position(self) -> str:
        - Read from db._datakeeper_cursor.find_one({"_id": "position"})
        - Default: "2024-01-01T00:00:00+00:00"

      async _set_cursor_position(self, timestamp: str):
        - Upsert to db._datakeeper_cursor with last_timestamp and updated_at

      async _is_processed(self, event_id: str, handler_name: str) -> bool:
        - Check db._datakeeper_processed for compound key (event_id, handler)

      async _mark_processed(self, event_id: str, handler_name: str):
        - Insert into db._datakeeper_processed with processed_at timestamp

      async _process_event(self, event: dict) -> bool:
        - Get handlers for event["event_type"]; if none, return True
        - Split into critical (sequential) and parallel lists
        - For each critical handler: check _is_processed first (idempotency), run handler, _mark_processed on success. On failure: return False immediately
        - For parallel handlers: asyncio.gather with return_exceptions=True. Log failures but don't block. Mark each successful one as processed.
        - Return True if all critical handlers succeeded

      async _send_to_dlq(self, event: dict, error: str = ""):
        - Read retry doc from _datakeeper_retries by event_id
        - Increment attempts
        - If attempts >= MAX_RETRIES: insert into dead_letter collection, delete from _datakeeper_retries
        - Else: upsert retry doc with incremented attempts and retry_at timestamp

      async run(self):
        - Set self._running = True, log startup
        - While running: poll roddos_events with timestamp $gt cursor, sort by timestamp asc, limit BATCH_SIZE
        - Also poll _datakeeper_retries for retry_at <= now (limit 10)
        - Process new events in order; advance cursor on success, DLQ on failure
        - Process retry events; delete retry doc on success, DLQ on failure
        - asyncio.sleep(POLL_INTERVAL_SECONDS)
        - Wrap in try/except to never crash the loop

      async stop(self):
        - Set self._running = False

      Property events_processed -> int (for status endpoint)
      Property last_event_timestamp -> str | None (for status endpoint)

    async def ensure_datakeeper_indexes(db: AsyncIOMotorDatabase):
      - Create index on roddos_events.timestamp
      - Create unique compound index on _datakeeper_processed (event_id, handler)
      - Create index on _datakeeper_retries.retry_at
      - Create index on dead_letter.dead_at

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| roddos_events -> EventProcessor | Events are internal (published by Contador), but datos payload must still be validated before acting on it |
| EventProcessor -> MongoDB collections | Handler writes must pass validate_write_permission() before touching business collections |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | event_handlers.py | mitigate | Validate event schema (event_type, datos fields present) before acting on datos values in every handler |
| T-07-02 | Denial of Service | event_processor.py | mitigate | MAX_RETRIES=3 + dead_letter collection; poison events can never cause infinite retry loop |
| T-07-03 | Elevation of Privilege | event_handlers.py | mitigate | Every handler calls validate_write_permission("loanbook", collection_name) before any MongoDB write to business collections |
| T-07-04 | Denial of Service | event_processor.py | mitigate | BATCH_SIZE=50 limit per poll cycle prevents processing unbounded events in one tick; asyncio.sleep(5) prevents tight loop |
| T-07-05 | Information Disclosure | routers/datakeeper.py | accept | Status and DLQ endpoints expose event metadata; acceptable for internal monitoring tool; no PII in event payloads |
</threat_model>
